# Compact Smart Contract - Project Scaffold

Create a new Compact smart contract project following Edda Labs standards.

## Arguments

- `$0` = contract name (kebab-case, e.g. `my-new-contract`)
- `$1` = short description of the contract purpose

## Project Structure

Every contract project follows this structure:

```
<contract-name>/
├── package.json
├── tsconfig.json
├── tsconfig.build.json
├── vitest.config.ts
├── eslint.config.mjs
├── .prettierrc
└── src/
    ├── <contract-name>.compact       # Main contract (imports all modules)
    ├── index.ts                      # Exports managed contract + witnesses
    ├── witnesses.ts                  # Private state + witness functions
    ├── config.ts                     # Logger config
    ├── logger.ts                     # Pino logging utility
    ├── managed/                      # Generated by compact compiler (never edit)
    │   └── <contract-name>/
    ├── modules/
    │   ├── access-control/           # Role-based access control
    │   │   ├── AccessControl.compact
    │   │   ├── Initializable.compact
    │   │   └── Pausable.compact
    │   ├── identity/                 # User verification whitelist
    │   │   ├── Identity.compact
    │   │   ├── Initializable.compact
    │   │   └── Pausable.compact
    │   ├── token-nft/                # NFT certificate management
    │   │   ├── NonFungibleToken.compact
    │   │   ├── Initializable.compact
    │   │   └── Pausable.compact
    │   ├── <custom-module>/          # Business-logic module (NEW per contract)
    │   │   ├── <Module>.compact
    │   │   ├── Initializable.compact
    │   │   └── Pausable.compact
    │   └── utils/
    │       ├── Utils.compact
    │       └── IdentityUtils.compact
    └── test/
        ├── <contract-name>.test.ts
        ├── simulators/
        │   └── simulator.ts
        └── utils/
            └── utils.ts
```

## Steps

1. **Create directory structure** - All directories as shown above
2. **Copy config files** from `bucket-defi-contract/` adapting:
   - `package.json`: change `name` to `@eddalabs/<contract-name>`, update `compact` and `build` scripts to reference the new `.compact` file and managed output path
   - `tsconfig.json`, `tsconfig.build.json`, `vitest.config.ts`, `eslint.config.mjs`, `.prettierrc`: copy as-is
3. **Copy TypeScript support files** adapting import paths:
   - `index.ts`: export from `./managed/<contract-name>/contract/index.js`
   - `witnesses.ts`: import Ledger from managed path
   - `config.ts`, `logger.ts`: copy as-is
4. **Copy standard modules** (access-control, identity, token-nft, utils) - all Initializable.compact and Pausable.compact files are identical across modules
5. **Modify AccessControl.compact** if the role hierarchy differs (see compact-module skill)
6. **Create the custom business-logic module** (see compact-module skill)
7. **Write the main contract** `.compact` file that orchestrates all modules
8. **Add to monorepo** `workspaces` array in root `package.json`
9. **Run `npm install`** at monorepo root
10. **Compile**: `npm run compact` in project directory
11. **Write tests** (see compact-test skill)

## Key Conventions

### Compact Language Basics

```compact
pragma language_version >= 0.20;
import CompactStandardLibrary;

// Module definition
module MyModule {
  import CompactStandardLibrary;
  import "./Initializable" prefix Initializable_;
  import "./Pausable" prefix Pausable_;

  export ledger myState: Map<Bytes<32>, Boolean>;
  export ledger myCounter: Counter;

  export circuit initialize(): [] {
    Initializable_initialize();
  }

  export circuit myCircuit(param: Uint<128>): [] {
    Initializable_assertInitialized();
    Pausable_assertNotPaused();
    // logic here
  }
}
```

### Main Contract Pattern

```compact
pragma language_version >= 0.20;
import CompactStandardLibrary;
import "./modules/access-control/AccessControl" prefix AccessControl_;
import "./modules/identity/Identity" prefix Identity_;
import "./modules/token-nft/NonFungibleToken" prefix NonFungibleToken_;
import "./modules/<custom>/<Module>" prefix Module_;

// Re-export public state from all modules
export { AccessControl__operatorRoles, AccessControl_adminRoles, AccessControl__ADMIN_MASTER };
export { Identity__verifiedUsersforBuckets };
// ... etc

// Constructor initializes ALL modules
constructor(_name: Opaque<"string">, _symbol: Opaque<"string">) {
  AccessControl_initialize();
  Identity_initialize();
  NonFungibleToken_initialize(_name, _symbol);
  Module_initialize();
}

// Wrapper circuits add cross-module logic + access control
export circuit myAction(param: Uint<128>): [] {
  assertOnlyRole(4 as Field as Bytes<32>);  // Check role
  assertOwnVerification();                   // Check identity
  // Cross-module state reads happen HERE at top level
  const data = NonFungibleToken__tokenToPrice.lookup(disclose(param));
  Module_doAction(param);
}
```

### Role Hierarchy Pattern

Roles are `Bytes<32>` values cast from integers. Two proven patterns exist:

**Flat hierarchy (Private Buyer pattern - preferred for simpler contracts):**
```
Admin         = 0 (default<Bytes<32>>)  -- manages all roles directly
Minter        = 1
PoolOperator  = 2
Verifier      = 3
```

**Two-tier hierarchy (Bucket DEFI pattern - for complex organizational structures):**
```
Master Admin  = 0 (default<Bytes<32>>)  -- manages admin roles
MinterAdmin   = 1    Minter        = 2
MatcherAdmin  = 3    Matcher       = 4
SettlerAdmin  = 5    Settler       = 6
VerifierAdmin = 7    Verifier      = 8
```

The flat hierarchy is simpler and cheaper (fewer role management transactions). Use the two-tier hierarchy only when delegation of role management is a requirement.

### Shielded Coin Patterns

```compact
// Receive a coin (buyer pays)
receiveShielded(disclose(coin));
assert(coin.value >= price as Uint<128>, "Insufficient payment");
assert(coin.color == nativeToken(), "Invalid coin");

// Store coin in a map (contract holds funds)
myMap.insertCoin(key, disclose(coin), right<ZswapCoinPublicKey, ContractAddress>(kernel.self()));

// Send coins to a recipient
const send_result = sendShielded(potFromMap, recipient, amount);
if (send_result.change.value.value != 0) {
  myMap.insertCoin(key, send_result.change.value, right<...>(kernel.self()));
} else {
  myMap.insert(key, default<QualifiedShieldedCoinInfo>);
}
```

### Seller Payment Accumulation Pattern (Private Buyer)

When multiple purchases accumulate balance for a seller:

```compact
// Two maps: one for coin info, one for numeric amount tracking
export ledger _sellerBalance: Map<Either<ZswapCoinPublicKey, ContractAddress>, QualifiedShieldedCoinInfo>;
export ledger _sellerAmount: Map<Either<ZswapCoinPublicKey, ContractAddress>, Uint<128>>;

// On purchase: store/accumulate payment
_sellerBalance.insertCoin(disclose(seller), disclose(coin), right<...>(kernel.self()));
if (_sellerAmount.member(disclose(seller))) {
  const current = _sellerAmount.lookup(disclose(seller));
  _sellerAmount.insert(disclose(seller), disclose((current + disclose(price)) as Uint<128>));
} else {
  _sellerAmount.insert(disclose(seller), disclose(disclose(price) as Uint<128>));
}

// On withdrawal: send accumulated amount using ownPublicKey() for caller identity
const caller = left<ZswapCoinPublicKey, ContractAddress>(ownPublicKey());
const balance = _sellerBalance.lookup(disclose(caller));
const amount = _sellerAmount.lookup(disclose(caller));
const send_result = sendShielded(balance, caller, amount);
_sellerAmount.insert(disclose(caller), 0);
```

### NFT Pool Pattern (Private Buyer)

Direct purchase pool where NFTs are listed by a PoolOperator and purchased by verified buyers:

```compact
export ledger _pool: Set<Uint<128>>;                                // Listed tokens
export ledger _tokenSold: Map<Uint<128>, Boolean>;                  // Sold tracking
export ledger _nftOwnerCommitment: Map<Uint<128>, Bytes<32>>;       // Buyer commitment per token

// List: PoolOperator adds token to pool
_pool.insert(disclose(tokenId));

// Purchase: remove from pool, mark sold, store commitment
_pool.remove(disclose(tokenId));
_tokenSold.insert(disclose(tokenId), true);
_nftOwnerCommitment.insert(disclose(tokenId), ownerCommitment);
```

### Batch Purchase Pattern (Private Buyer)

Multiple NFTs purchased with a single shared commitment. Uses mock tokens (tokenId == 0) for partial batches:

```compact
// Skip mock tokens at every step
export circuit _getBatchTokenPrice(tokenId: Uint<128>): Uint<64> {
  if (disclose(tokenId) != 0) {
    return NonFungibleToken__tokenToPrice.lookup(disclose(tokenId));
  }
  return 0;
}

// Compute ONE commitment for all tokens in the batch
const ownerCommitment = NFTPool_computeBatchCommitment();

// Execute purchase per real token (shares same commitment)
_executeBatchPurchase(tokenId1, ownerCommitment);
_executeBatchPurchase(tokenId2, ownerCommitment);
// ... up to batch size
```

### Burn Guards Pattern (Private Buyer)

Protect burn from destroying active tokens:

```compact
export circuit burn(tokenId: Uint<128>): [] {
  assertOnlyRole(1 as Field as Bytes<32>);  // Minter role
  assert(!NFTPool__pool.member(disclose(tokenId)), "Token must be unlisted before burning");
  if (NFTPool__tokenSold.member(disclose(tokenId))) {
    assert(!NFTPool__tokenSold.lookup(disclose(tokenId)), "Sold tokens must be burned by commitment owner");
  }
  return NonFungibleToken_burn(tokenId);
}
```

### Internal Function Access Pattern

When the top-level contract needs to call a module's internal `_function` (bypassing the module's own access checks because access control is handled at the top level):

```compact
// Module defines internal function with underscore prefix:
//   export circuit _setTokenPrice(tokenId, price): [] { ... }

// Top-level import: import "./modules/token-nft/NonFungibleToken" prefix NonFungibleToken_;

// Calling _setTokenPrice becomes NonFungibleToken_ + _setTokenPrice = NonFungibleToken__setTokenPrice
// (double underscore: import prefix + underscore from function name)

export circuit setTokenPrice(tokenId: Uint<128>, price: Uint<64>): [] {
  assertOnlyRole(2 as Field as Bytes<32>);  // PoolOperator checks access
  NonFungibleToken__setTokenPrice(tokenId, price);  // Bypass module's owner check
}
```

### ZK Ownership Commitment Pattern

```compact
// Compute owner ID from public key + secret nonce
export witness wit_secretNonce(): Bytes<32>;

export pure circuit _computeOwnerId(pk: Either<ZswapCoinPublicKey, ContractAddress>, nonce: Bytes<32>): Bytes<32> {
  assert(pk.is_left, "Contract address owners not supported");
  return persistentHash<Vector<2, Bytes<32>>>([pk.left.bytes, nonce]);
}

// Compute commitment = hash(ownerId, counter, domainSeparator)
export circuit _computeOwnerCommitment(id: Bytes<32>, counter: Uint<64>): Bytes<32> {
  return persistentHash<Vector<3, Bytes<32>>>(
    [id, counter as Field as Bytes<32>, pad(32, "MyContract: shield:")]);
}
```
