# Compact Smart Contract - Project Scaffold

Create a new Compact smart contract project following Edda Labs standards.

## Arguments

- `$0` = contract name (kebab-case, e.g. `my-new-contract`)
- `$1` = short description of the contract purpose

## Project Structure

Every contract project follows this structure:

```
<contract-name>/
├── package.json
├── tsconfig.json
├── tsconfig.build.json
├── vitest.config.ts
├── eslint.config.mjs
├── .prettierrc
└── src/
    ├── <contract-name>.compact       # Main contract (imports all modules)
    ├── index.ts                      # Exports managed contract + witnesses
    ├── witnesses.ts                  # Private state + witness functions
    ├── config.ts                     # Logger config
    ├── logger.ts                     # Pino logging utility
    ├── managed/                      # Generated by compact compiler (never edit)
    │   └── <contract-name>/
    ├── modules/
    │   ├── access-control/           # Role-based access control
    │   │   ├── AccessControl.compact
    │   │   ├── Initializable.compact
    │   │   └── Pausable.compact
    │   ├── identity/                 # User verification whitelist
    │   │   ├── Identity.compact
    │   │   ├── Initializable.compact
    │   │   └── Pausable.compact
    │   ├── token-nft/                # NFT certificate management
    │   │   ├── NonFungibleToken.compact
    │   │   ├── Initializable.compact
    │   │   └── Pausable.compact
    │   ├── <custom-module>/          # Business-logic module (NEW per contract)
    │   │   ├── <Module>.compact
    │   │   ├── Initializable.compact
    │   │   └── Pausable.compact
    │   └── utils/
    │       ├── Utils.compact
    │       └── IdentityUtils.compact
    └── test/
        ├── <contract-name>.test.ts
        ├── simulators/
        │   └── simulator.ts
        └── utils/
            └── utils.ts
```

## Steps

1. **Create directory structure** - All directories as shown above
2. **Copy config files** from `bucket-defi-contract/` adapting:
   - `package.json`: change `name` to `@eddalabs/<contract-name>`, update `compact` and `build` scripts to reference the new `.compact` file and managed output path
   - `tsconfig.json`, `tsconfig.build.json`, `vitest.config.ts`, `eslint.config.mjs`, `.prettierrc`: copy as-is
3. **Copy TypeScript support files** adapting import paths:
   - `index.ts`: export from `./managed/<contract-name>/contract/index.js`
   - `witnesses.ts`: import Ledger from managed path
   - `config.ts`, `logger.ts`: copy as-is
4. **Copy standard modules** (access-control, identity, token-nft, utils) - all Initializable.compact and Pausable.compact files are identical across modules
5. **Modify AccessControl.compact** if the role hierarchy differs (see compact-module skill)
6. **Create the custom business-logic module** (see compact-module skill)
7. **Write the main contract** `.compact` file that orchestrates all modules
8. **Add to monorepo** `workspaces` array in root `package.json`
9. **Run `npm install`** at monorepo root
10. **Compile**: `npm run compact` in project directory
11. **Write tests** (see compact-test skill)

## Key Conventions

### Compact Language Basics

```compact
pragma language_version >= 0.20;
import CompactStandardLibrary;

// Module definition
module MyModule {
  import CompactStandardLibrary;
  import "./Initializable" prefix Initializable_;
  import "./Pausable" prefix Pausable_;

  export ledger myState: Map<Bytes<32>, Boolean>;
  export ledger myCounter: Counter;

  export circuit initialize(): [] {
    Initializable_initialize();
  }

  export circuit myCircuit(param: Uint<128>): [] {
    Initializable_assertInitialized();
    Pausable_assertNotPaused();
    // logic here
  }
}
```

### Main Contract Pattern

```compact
pragma language_version >= 0.20;
import CompactStandardLibrary;
import "./modules/access-control/AccessControl" prefix AccessControl_;
import "./modules/identity/Identity" prefix Identity_;
import "./modules/token-nft/NonFungibleToken" prefix NonFungibleToken_;
import "./modules/<custom>/<Module>" prefix Module_;

// Re-export public state from all modules
export { AccessControl__operatorRoles, AccessControl_adminRoles, AccessControl__ADMIN_MASTER };
export { Identity__verifiedUsersforBuckets };
// ... etc

// Constructor initializes ALL modules
constructor(_name: Opaque<"string">, _symbol: Opaque<"string">) {
  AccessControl_initialize();
  Identity_initialize();
  NonFungibleToken_initialize(_name, _symbol);
  Module_initialize();
}

// Wrapper circuits add cross-module logic + access control
export circuit myAction(param: Uint<128>): [] {
  assertOnlyRole(4 as Field as Bytes<32>);  // Check role
  assertOwnVerification();                   // Check identity
  // Cross-module state reads happen HERE at top level
  const data = NonFungibleToken__tokenToPrice.lookup(disclose(param));
  Module_doAction(param);
}
```

### Role Hierarchy Pattern

Roles are `Bytes<32>` values cast from integers. Each operator role has an admin role:

```
Master Admin  = 0 (default<Bytes<32>>)
MinterAdmin   = 1    Minter        = 2
CustomAdmin   = 3    CustomRole    = 4
VerifierAdmin = 5    Verifier      = 6
```

### Shielded Coin Patterns

```compact
// Receive a coin (buyer pays)
receiveShielded(disclose(coin));
assert(coin.value >= price as Uint<128>, "Insufficient payment");
assert(coin.color == nativeToken(), "Invalid coin");

// Store coin in a map (contract holds funds)
myMap.insertCoin(key, disclose(coin), right<ZswapCoinPublicKey, ContractAddress>(kernel.self()));

// Send coins to a recipient
const send_result = sendShielded(potFromMap, recipient, amount);
if (send_result.change.value.value != 0) {
  myMap.insertCoin(key, send_result.change.value, right<...>(kernel.self()));
} else {
  myMap.insert(key, default<QualifiedShieldedCoinInfo>);
}
```

### ZK Ownership Commitment Pattern

```compact
// Compute owner ID from public key + secret nonce
export witness wit_secretNonce(): Bytes<32>;

export pure circuit _computeOwnerId(pk: Either<ZswapCoinPublicKey, ContractAddress>, nonce: Bytes<32>): Bytes<32> {
  assert(pk.is_left, "Contract address owners not supported");
  return persistentHash<Vector<2, Bytes<32>>>([pk.left.bytes, nonce]);
}

// Compute commitment = hash(ownerId, counter, domainSeparator)
export circuit _computeOwnerCommitment(id: Bytes<32>, counter: Uint<64>): Bytes<32> {
  return persistentHash<Vector<3, Bytes<32>>>(
    [id, counter as Field as Bytes<32>, pad(32, "MyContract: shield:")]);
}
```
